📌 Decision — Agent Architecture

Choice

Two-tier agent storage, never mixed.

The Two Tiers

| Path | Audience | Purpose | Tracked |
|------|----------|---------|:-------:|
| .claude/agents/ | Boss-facing | Project helpers | git |
| ~/.hermes/profiles/axis/agents/engineering/ | AXIS internals | Engineering force | Hermes config |

Why

  • Boss-facing agents may go in commits (project-specific)
  • Internal agents are AXIS-only, may use Hermes features
  • Mixing causes confusion + leaks
  • Cross-profile write guard refuses by default

Rules

  • When Boss asks for "an agent", default to .claude/agents/ if project work
  • For AXIS internal work, use ~/.hermes/profiles/axis/agents/
  • Never copy one to the other without explicit Boss direction
  • Pass cross_profile=True only with explicit permission

Common Commands


pm2 list                    # all services

Recovery


# Service down
pm2 restart <app>

# Service won't start
pm2 logs <app> --err --lines 50
# Check: EADDRINUSE? port collision? env missing?

# Full recovery
pm2 delete <app>
cd /root/<project> && pnpm start &
pm2 save

# Auto-start on reboot
pm2 startup
pm2 save

Disk Full


df -h
pm2 flush                # clear logs
journalctl --vacuum-time=7d

curl -I https://<domain>
